# CYCLE_ROLLOVER_CHECKLIST

Use this checklist when moving from one cycle to the next (for example, 2026 -> 2027).

## 1) Obtain and Stage the Raw Survey Export

Receive the new cycle's raw CSV from the survey provider.

Naming convention:
```
OpsSurveyRawData_YYYY.csv
```
Example: `OpsSurveyRawData_2027.csv`

Place the file in `data/`.

> [!TIP]
> The exact filename doesn't matter to the pipeline (it's tracked in `config/cycles.csv`), but using a consistent naming convention makes the project easier to navigate.

## 2) Prepare Required Metadata Files

Required in `data/`:
- `response_key_YY.csv`
- `progress_YY.csv`

These are cycle input metadata files. They are not auto-generated from the raw survey CSV.

Start by copying the prior cycle's versions, renaming them for the new cycle, and updating for any survey changes. See `METADATA_FORMAT_REFERENCE.md` for the exact column specifications and valid values.

Important distinction:
- `response_key_YY.csv` is the maintained input file for the cycle.
- `response_key.csv` without the year suffix is only a pipeline-generated snapshot written after a successful run.

If the survey questions changed, follow `SURVEY_CHANGES_GUIDE.md` before proceeding.

## 3) Prepare Optional Override Files

Optional override files (cycle-scoped) in `data/`:
- `institution_overrides_YY.csv`
- `value_overrides_employer_YY.csv`
- `value_overrides_budget_YY.csv`

Create empty files with just headers if no overrides are needed for the new cycle. See `MAINTENANCE_NOTES.md` §1 for column specs.

## 4) Add a New Row in `config/cycles.csv`

Required row fields:
- `cycle_id`
- `cycle_label`
- `cycle_subtitle`
- `raw_data_file`
- `response_key_file`
- `progress_file`
- `schema_report_file`
- `institution_overrides_file`
- `employer_overrides_file`
- `budget_overrides_file`
- `rdata_file`
- `general_report_rmd`
- `custom_report_rmd`
- `index_rmd`
- `custom_output_dir`

### 2027 Template Row (Copy/Edit)

Add this line to `config/cycles.csv` and update filenames as needed:

```csv
27,2026-2027,2026-27 LACN Operations Survey,data/OpsSurveyRawData_2027.csv,data/response_key_27.csv,data/progress_27.csv,data/schema_check_27.csv,data/institution_overrides_27.csv,data/value_overrides_employer_27.csv,data/value_overrides_budget_27.csv,lacn.RData,docs/GeneralReport.Rmd,docs/custom_template.Rmd,index.Rmd,docs/custom
```

Quick verification:

```bash
sed -n '1,5p' config/cycles.csv
```

## 5) Optional: Set Publication Label

The landing page now derives cycle year and response count from the active workspace automatically.

If the publish month/year should differ from the current date, set:

```bash
export LACN_PUBLICATION_LABEL="March 2027"
```

## 6) Run New-Cycle Build

From `lacn2025/`:

```bash
Rscript code/run_cycle.R --cycle=YY
```

If it fails at schema stage, inspect:
- `data/schema_check_YY.csv`
- Follow `TROUBLESHOOTING.md` §2 for resolution steps

## 7) Run Blocking Gates

Follow:
- `VALIDATION_GATES.md`

## 8) Rollover Simulation Test

Before real release, run a dry simulation:
1. Add mocked cycle row `cycle_id=YY`.
2. Place mocked `response_key_YY.csv` and `progress_YY.csv` in `data/`.
3. Run `Rscript code/run_cycle.R --cycle=YY`.
4. Confirm no undocumented manual step is needed.
5. Update this checklist if any hidden step was discovered.

Cycle-scoped validation outputs (generated by the pipeline):
- `data/schema_check_YY.csv`
- `data/cycle_validation_YY.csv`
- `data/stolaf_validation_YY.csv`
